# EDA メモ

## 全体傾向

学習データの平均売上は右に裾が長く、カテゴリや曜日の影響が強いです。単純な全体平均だけでは説明しきれないため、`store_nbr` と `family`、さらに曜日や販促の情報を使う方針が妥当でした。

## 曜日パターン

曜日別平均売上を見ると、週末に売上が高くなる傾向が確認できました。特に土曜・日曜が強く、平日との差が大きいです。

![曜日別平均売上](../reports/sales_by_dow.png)

## 販促の影響

`onpromotion` が大きいほど平均売上は上がる傾向がありました。完全な時系列モデルを作る前段として、販促をそのまま説明変数に入れる方針は有効です。

![販促フラグ別平均売上](../reports/sales_by_promo_flag.png)

## モデル選択への反映

- 直近の売上を逐次予測するラグモデルは、今回の holdout では安定しませんでした。
- 代わりに、`store_nbr`, `family`, `dayofweek`, `onpromotion` の階層平均が最も安定して改善しました。
- 祝日や油価は探索対象として残していますが、今回の最終案では採用していません。
